# 베프(BEFF) 사용자 앱 발표 시연 스크립트

## 핵심 메시지

베프 사용자 앱은 장애학생과 비장애학생이 한 화면에서 캠퍼스의 물리적 장벽을 낮추고, 자연스러운 친구 매칭으로 사회적 고립을 줄이며, 일상의 안전망(SOS)을 함께 가지도록 만든 무장애 교류 플랫폼입니다.

## 3분 시연 흐름

| 순서 | 화면 | 말할 내용 |
|---:|---|---|
| 1 | 홈 | "오늘의 함께하는 활동 배너, 인기 관심 분야, 마감 임박 활동을 한 화면에 모았습니다." |
| 2 | 홈 — SOS 안전 카드 | "어떤 화면에서도 1탭으로 긴급 도움을 부를 수 있습니다." |
| 3 | 지도 | "캠퍼스 무장애 시설을 카테고리·접근 가능 여부로 필터링하고, 사용자가 직접 신고할 수 있습니다." |
| 4 | 매칭 | "이용자/제공자가 같은 카드 위에서 만납니다. '완료 시 봉사 4시간 인정' 같은 인센티브가 카드에 같이 노출됩니다." |
| 5 | 프로필 | "비장애학생에게는 봉사시간·활동확인서·학점 연계가, 장애학생에게는 접근 프로파일과 안전망이 같은 위계로 노출됩니다." |
| 6 | SOS 다이얼로그 | "호출 대상, 메시지, 위치가 한 번에 전송되고 응답 시간이 기록됩니다." |

## 5분 시연 흐름

1. **홈**
   - 헤더 알림 배지, 검색바, 히어로 배너(좌우 스와이프), 인기 관심 분야 TOP 5(드래그 스크롤), 빠른 메뉴 8개를 보여줍니다.
   - "포용 기능(배리어프리/동행매칭)을 학식·셔틀과 동등 위계로 배치한 점이 핵심입니다."

2. **SOS 안전 카드 → 다이얼로그**
   - 카드 탭 → 전체화면 다이얼로그.
   - 호출 대상(보호자/지원센터/보안실/119) 선택, 빠른 메시지 칩 복수 선택, 위치 자동 감지를 보여줍니다.
   - "현재는 Mock 동작이며, 실제 호출은 백엔드 추가 필요입니다."

3. **지도**
   - 카테고리 필터(엘리베이터/경사로/화장실/건물), 마커 클릭 → 하단 시트 상세, 길찾기/저장 버튼.
   - "공공데이터(전국장애인편의시설표준데이터, BF 인증)를 기초 레이어로 두고 사용자 신고로 보완하는 구조입니다."

4. **매칭**
   - "동행 요청 / 동행 제공 / 내 매칭" 탭.
   - 카드 위 신뢰 배지(학교 이메일 인증, 학교 공동 인증 활동), 매너 점수, '완료 시 봉사 N시간 인정' 배너.
   - 신청 다이얼로그.
   - 내 매칭 탭의 5단계 스텝퍼(접수→매칭→일정조율→동행→종료/설문).

5. **프로필**
   - 신뢰 배지(이메일 인증, MOU, 매너 점수).
   - **봉사시간·활동확인서 카드** (누적/이번 학기/목표 진행률, 발급 다이얼로그, 학점 신청 안내).
   - **접근 프로파일** (필요한 도움 / 제공 가능한 도움).
   - 친구 가로 스크롤 → 전체보기 다이얼로그.
   - **접근성 설정** 다이얼로그(글자 크기 슬라이더 미리보기, 고대비/모션 줄이기/자막/수어/음성 해설).

6. **마무리 메시지**
   - "이 앱 하나로 장애학생은 안전망과 친구를, 비장애학생은 봉사시간과 학점 연계를, 학교는 운영 데이터를 동시에 얻습니다."

## 발표 때 조심할 표현

- "연동되어 있습니다"라고 말하지 않습니다.
- "현재는 Mock 데이터로 시연하고, 실제 호출·발급은 백엔드 추가 필요입니다"라고 설명합니다.
- "AI가 매칭합니다"보다 "접근 프로파일을 기반으로 매칭 후보를 추천하고 사용자가 선택합니다"라고 설명합니다.
- "공공데이터가 완벽합니다"보다 "공공데이터를 기초로 하고 현장 사용자 신고로 보완합니다"라고 설명합니다.
- SOS에 대해 "자동으로 신고됩니다"라고 단정하지 않고 "선택한 대상에게 위치와 메시지가 전송됩니다"라고 설명합니다.
