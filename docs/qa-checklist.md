# 베프(BEFF) 사용자 앱 QA 체크리스트

## 빌드 확인

- [ ] `npm install`
- [ ] `npm run build`
- [ ] Vite 빌드 성공
- [ ] TypeScript 에러 없음

## 로컬 실행 확인

```bash
npm run dev
```

기본 주소: `http://localhost:5173/`

- [ ] 홈 탭 (기본)
- [ ] 지도 탭
- [ ] 매칭 탭 — 동행 요청 / 동행 제공 / 내 매칭
- [ ] 프로필 탭
- [ ] SOS FAB → 다이얼로그 열림/닫힘
- [ ] 프로필 → 접근성 설정 다이얼로그
- [ ] 프로필 → 활동확인서 다이얼로그
- [ ] 프로필 → 친구 전체보기 다이얼로그
- [ ] 매칭 → 신청 다이얼로그
- [ ] 매칭 → 동행 등록 다이얼로그

## 화면 QA

- [ ] 하단 네비게이션이 스크롤 시 흔들리지 않는가?
- [ ] SOS FAB이 모든 탭에서 우하단에 떠 있는가?
- [ ] 매칭 카드의 "완료 시 봉사 N시간 인정" 배너가 이용자 카드에 노출되는가?
- [ ] 프로필 봉사시간 진행률이 올바르게 채워지는가?
- [ ] 접근성 설정의 글자 크기 슬라이더 미리보기가 즉시 반영되는가?
- [ ] 지도 마커가 선택 시 색·크기·zIndex 변하는가?
- [ ] 지도 하단 시트가 선택/목록 모드로 토글되는가?
- [ ] 매칭 스텝퍼가 진행 단계에 맞게 활성화되는가?
- [ ] 콘솔 에러가 없는가?

## 접근성 QA (사용자 앱 핵심)

- [ ] 모든 인터랙티브 요소가 44px 이상 터치 영역인가?
- [ ] 각 버튼·링크·아이콘에 `aria-label` 또는 명시적 텍스트가 있는가?
- [ ] 헤딩 구조(h1, h2 …)가 페이지별 1회씩 들어 있는가?
- [ ] 포커스 이동 시 `focus-visible` 외곽선이 보이는가?
- [ ] 마커·아이콘 등 장식 요소에 `aria-hidden="true"`가 있는가?
- [ ] 매칭 상태 변경 등 동적 영역에 `aria-live`가 있는가?
- [ ] iOS Safari에서 하단 safe-area-inset이 적용되는가?
- [ ] 색만으로 정보 전달하는 경우(상태/배지)가 없는가? (아이콘+텍스트 동반 확인)

## 반응형 확인

- [ ] 360px 모바일 폭(좁은 안드로이드)
- [ ] 390px 모바일 폭(iPhone)
- [ ] 768px 태블릿
- [ ] 1024px+ 데스크톱 (사이드 여백 처리)

사용자 앱은 모바일 우선 화면입니다. 좁은 화면에서 텍스트 줄바꿈·버튼 정렬이 깨지지 않아야 합니다.

## 보안/업로드 전 확인

- [ ] `.env` 파일이 GitHub에 올라가지 않는가?
- [ ] API Key, Secret, Token이 코드에 하드코딩되어 있지 않은가?
- [ ] `node_modules/`가 올라가지 않는가?
- [ ] `dist/`가 별도 요청 없이 올라가지 않는가?
- [ ] 실제 사용자 개인정보가 포함되어 있지 않은가?
- [ ] 실제 학교 내부 데이터처럼 보이는 문구가 없는가?
- [ ] SOS 호출 대상의 실제 전화번호가 노출되어 있지 않은가? (Mock은 `***` 처리)

## 추천 검색 명령

```bash
rg -n "API_KEY|SECRET|TOKEN|PASSWORD|BEGIN PRIVATE|sk-|ghp_|xoxb-|DB_PASSWORD|\\.env" . \
  --glob '!node_modules/**' \
  --glob '!dist/**' \
  --glob '!*.tsbuildinfo' \
  --glob '!.git/**'
```

검색 결과가 나오면 실제 비밀값인지, 문서상의 주의 문구인지 구분해서 확인합니다.
